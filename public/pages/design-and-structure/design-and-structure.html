Design and Structure

Designing an app it is important to follow certain agreed-upon practices, in order to facilitate ease of maintenance

App structure

When speaking of the structure of the app, we are focusing on where the different functions of the application can be found. It is important that anybody familiar with the node framework can sit down and work with a minimum of time spent figuring out the location of methods, and imports. As a rule in the app.js file you will find a standard structure across all node applications.

  • Imports - in the top, and instantiation of express
    •     
        import express from 'express';
        import templateEngine from './utils/templateEngine.js';

        const app = express();
        app.use(express.static('public'));
  • Pages - second
    •     
        const page = templateEngine.readPage('./public/pages/page/page.html');
        const pagePage = templateEngine.renderPage(page, {
          tabTitle: 'something | welcome'
        });
        
        app.get('/page', ((req, res) => {
          res.send(pagePage);
        }));
  • APIs - third
    •     
            app.get("/api/something", (req, res) => {
            res.send({data: thing});
            });
  • PORT - last where the open port is displayed
    •     
        const PORT = NUMBER;
        app.listen(PORT, (error) => {
        if (error) {
        console.log(error);
        }
        console.log('Server is running on port:', PORT);
        });
